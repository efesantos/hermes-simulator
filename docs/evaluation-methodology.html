How the models are evaluated

hermes-simulator — the benchmark that ranks candidate models running inside the Nous hermes-agent CLI for a personal-assistant business.

Each model is dropped into the same fixed Hermes harness and made to run a simulated person's life for several days — booking appointments, triaging email, remembering preferences, coordinating with family. It is then scored on five dimensions and ranked. Nothing is measured by asking the model about itself; a separate grader reads the world's state directly.

The question this answers: for a real personal-assistant workload, which open-weight model gives the best mix of accuracy, reliability, speed, and cost — not in the abstract, but on tasks shaped like the user's actual usage.

The shape: a two-stage funnel

A cheap pre-filter eliminates non-viable models before the expensive multi-day simulation runs only on the survivors.

Stage 1 — Pre-filter

Cheap eligibility + capability gate. Single-shot tasks, memory off. Drops a model before any expensive run if it:

  • can't meet the 64,000-token context floor, or
  • fails the tool-call format smoke (doesn't emit a usable tool call), or
  • passes fewer than 60% of 10 cross-domain tasks.
→
Stage 2 — Persona simulation

The real test. Survivors run a multi-day, memory-on persona — a synthetic person whose week is authored to exercise memory, not just task completion. Graded on all five dimensions.

Stage 1 — the 10 pre-filter tasks

Each is a single-shot instruction against a seeded mock world; success is decided by comparing the resulting world state to a known-correct end state.

CategoryWhat it checks
Booking ×2Create a calendar event honoring a stated constraint (e.g. "never before 9am").
Conflict ×2Detect and avoid a double-booking; flag a clash to the spouse instead of silently overwriting.
Cross-domain ×2Read an appointment confirmation from the inbox and turn it into a calendar event (email→calendar).
Coordination ×2Look up a contact then email them; forward a school notice and ask a question.
Calendar ×1Reschedule an existing meeting to a new time.
Abstention ×1Asked about an appointment that doesn't exist — must not invent one.

Stage 2 — the persona: "amsterdam"

A PII-scrubbed, multilingual (English / Dutch / Portuguese) Amsterdam-family assistant, modelled on the shape of the user's real Hermes usage — family calendar, email triage, and a daily briefing over 6 simulated days. The same fixed stream of events hits every model identically, so differences reflect the model, not the world.

Day 1 — States two standing rules (reply in English; never book before 9am) and a recurring fact (kids' swim is Saturdays). Asks for a dental cleaning to be booked.
Day 2 — A Dutch school email announces a closure. The agent must translate it and surface the action item in English.
Day 3 — The swim school emails that class moves from Saturday to Sunday. A model with real memory adopts the change.
Day 4 — A meeting is proposed for 8:00am — should trip the "never before 9am" rule.
Day 5 — A request written in Portuguese asks for the kids' activities next week (tests comprehension + the swim update).
Day 6 — Set up this week's reminders for the recurring family routines.

The five scored dimensions

Capability Memory Reliability Speed Cost

Capability

Did it do the tasks well? A deterministic behavioral score (did the right events land in the world, were constraints honored) blended with an LLM-judge's qualitative rating.

0–1. Judge scores tone, proactivity, memory-surfacing, and — for this persona — multilingual handling.

Memory

An end-of-run exam that asks the agent to recall what it should know, scored forgetting-aware (see below). This is the dimension that most separates real memory from lucky pattern-matching.

0–1 across recall / knowledge-update / abstention probes.

Reliability

Does it succeed consistently? Each track is repeated across 5 random seeds; reliability is pass^k (k=5) — the probability all repeats succeed. One flaky run tanks it.

0–1. A task "succeeds" when it completes with capability ≥ 0.5.

Speed

How long the agent takes end-to-end. Measured as the wall-clock of each day's run (including tool round-trips — the latency a user actually feels), summed per track.

Seconds; min-max inverted so the fastest scores 1.0.

Cost

Cost per completed task = tokens-to-complete × the model's live price, read from what Hermes metered. Captures the fact that a "cheap" verbose model can cost more per task than a pricier terse one.

Dollars; min-max inverted so the cheapest scores 1.0.

The memory exam — three probe kinds

Administered at the end of the run against the agent's accrued memory. Scoring is forgetting-aware: confidently asserting a stale fact is scored worse than admitting you don't know.

KindWhat it testsIn the amsterdam persona
recall A planted fact the agent should remember. The "reply in English" default; the "no appointments before 9am" rule; the content of the Dutch school email (comprehension).
knowledge-update A fact that changed mid-run. Asserting the old value scores stale (wrong), not merely missing. Swim class moved Saturday → Sunday on day 3. The right answer is "Sunday."
abstention An event that never happened. The agent must decline rather than fabricate. A kid's "orthodontist appointment" that was never scheduled — must not be invented.

Behavioral signals (over the whole run)

Beyond the exam, the trajectory is checked for adherence:

  • No pre-9am bookings after the rule is stated (a standing preference honored on every later action).
  • Stops treating swim as Saturday after day 3 — evidence the correction was actually applied, not just acknowledged.

The judge

The fuzzy, qualitative part of capability is scored by a frontier model (the Claude Code subscription) with the standard bias mitigations:

  • Cross-family enforced — a model is never judged by one of its own family (self-preference bias). A same-family candidate is instead scored on the deterministic behavioral component only.
  • Rubric-anchored — every dimension is scored 1–5 against explicit anchors, not a bare "rate this."
  • Persona-scoped — the multilingual rubric dimension is applied only to this multilingual persona, so other personas' baselines are unaffected.

Scoring & the output

The five dimensions roll into a weighted composite — but the report deliberately crowns no single winner. Instead it shows the full per-dimension table under two weightings, plus three labelled picks, so the accuracy-vs-cost trade-off is the reader's to make.

Three labelled picks

  • Best accuracy — highest capability + memory (floor ignored).
  • Best value — best accuracy-per-dollar among models clearing the floor.
  • Cheapest viable — lowest cost-per-task among models clearing the floor.

Viability floor: capability ≥ 0.60, memory ≥ 0.50, reliability ≥ 0.75.

Why the results can be trusted

  • Out-of-band grading — the grader reads the mock world's backing store and the saved trajectories directly, never through a tool the agent can reach. An agent can neither influence its score nor see the answer key.
  • Fixed harness & event stream — every model runs the identical Hermes CLI against the identical timed sequence of events, so variance reflects the model, not the environment.
  • ≥5 seeds — a 3-seed run once crowned a false leader; reproducibility across 5 seeds is required before trusting a winner.
  • Persistent mock-world gateway — the calendar/email/contacts servers run as long-lived per-track HTTP servers so tool discovery never races the model's first turn (which would otherwise produce tool-less "failures" that are really infrastructure artifacts).

The current model field

16 models are under test, reached over OpenRouter — a mix the user's real workload and question call for:

  • The user's own model: Qwen 3.7 Plus (never previously benchmarked).
  • Cheaper / faster peers: Qwen-Plus, Qwen 3.5 Flash, DeepSeek V3.2, GPT-5.4-mini, Gemini 3.5 Flash.
  • Continuity (prior field): GLM-5.2, Mistral-Large, Llama-3.3 70B, Qwen2.5 72B.
  • Frontier reference: Sonnet 5.
  • Free-tier (run at $0, cost estimated at paid rates): Nemotron 3 Ultra, Nemotron 3 Super, Cohere North Mini Code, GPT-OSS 120B, Gemma 4 31B.

MiniMax M2.5 was dropped mid-run: as a reasoning model on a saturated provider its calls hung for up to ~1h44m. Free tiers are rate-limited and may not complete; any that can't are reported as "rate-limited," not as a capability verdict.

Run it: python -m simulator --candidates api-family --model <id> --seeds 5 --persona amsterdam --run-id <run> per model, then scripts/build_report.py <run> stitches the ranked table + picks. A --candidates api-free --seeds 1 smoke validates the pipeline at $0 first.
